# ポケット獣急外来（試作）

犬猫の救急初期対応と臨床推論を学ぶ教育シミュレーター。全症例共通の初療室から、必要な問診・検査・処置を選んで対応します。良し悪しはプレイ中には出ず、最終評価で初めて分かります。

- 依存ゼロの**単一HTMLファイル**（`index.html`）。ビルド不要、オフライン動作可。
- 症例：猫の尿道閉塞＋高K血症 / 犬のGDV / 猫の呼吸困難（メニューは全症例共通）。
- スマホ全画面レイアウト対応。

> ⚠️ 教育用シミュレーションです。用量・手技・実際の診療判断は、最新のガイドライン（RECOVER等）・添付文書・施設プロトコル・指導医に従ってください。投与量計算は本アプリでは扱いません。

## 動かす：一番かんたん（ローカル）

`index.html` をダブルクリックしてブラウザで開くだけ。

## Webアプリとして公開する

### 方法A：GitHub Pages（無料・URLが残る）

1. GitHub にログイン →右上「＋」→ **New repository** でリポジトリを作成（例：`pocket-vet-er`）。Public のままでOK。
2. リポジトリ画面の **Add file → Upload files** で、この `index.html`（必要なら `README.md` も）をドラッグして **Commit changes**。
3. **Settings → Pages** を開く。
4. *Build and deployment* の *Source* を **Deploy from a branch** にし、*Branch* を **main** ／ **/(root)** にして **Save**。
5. 1〜2分待つと、同じ Pages 画面の上部に公開URL（`https://<ユーザー名>.github.io/pocket-vet-er/`）が表示される。スマホでそのURLを開けばそのまま遊べます。

> 更新したいときは、新しい `index.html` を同じ手順でアップロード（上書きコミット）すれば自動で反映されます。

### 方法B：Netlify（ドラッグ＆ドロップが最速）

1. https://app.netlify.com にログイン。
2. **Add new site → Deploy manually**（または「Sites」画面に直接ドラッグ）。
3. この `index.html` が入ったフォルダをドロップするだけで即公開。発行されたURLをスマホで開けます。
4. 独自ドメインや再デプロイも同画面から可能。

Vercel / Cloudflare Pages も同様に静的サイトとして無料で公開できます。

## ランキングや保存を付けたくなったら

現状はバックエンド不要（手元で完結）。「みんなのスコアを保存してランキング」「ログイン」を付けたくなったら、その時に **Supabase**（DB＋認証が一体）か **Railway ＋ DB** を足します。それまでは静的ホスティングだけで完結します。

## ホーム画面に追加（スマホでアプリっぽく）

公開URLをスマホのブラウザで開き、共有メニューから「ホーム画面に追加」すると、全画面のアプリのように起動できます。

## 拡張のしかた（開発メモ）

`index.html` 内の JavaScript で、データとロジックを分離しています。

- **共通カタログ**：`CATALOG`（安定化／薬剤棚／検査／処置。薬剤・検査は薬効群・検査群でグループ化）、`HX_GROUPS`（問診）、`EXAM_GROUPS`（身体診察）、`DIAGNOSES`、`DISPOSITIONS`。全症例で共通表示されます。
- **症例ルール**：`CASES` の各症例が `res{ actionId → { text, s, time, reveal, flag, need, contra, tag, verdict, miss } }` を持ち、**選択後の結果・採点・状態変化・開示情報だけ**を定義。カタログに無い行為は中立の `defaultResult` を返します。
- 病態を示すラベルは入れず、一般的な医療行為名のみ。`need`（前提フラグ）・`contra`（禁忌フラグ）で順序ミス・禁忌を減点。検査するまで K/Cre 等は非表示。

**症例を追加するには**：`CASES` に新しいエントリを足し、`hidden`（隠しバイタル/検査値）、`physiology(S)`（時間・治療による状態変化）、`res`（関連行為の結果）、`correctDx` / `correctDispo` / `finalNote` を定義するだけ。メニュー側は変更不要です。
